二、非常重要：PyEIT 不直接支持把“电势矩阵”直接拿来重建
--------------------------------------------------

请你先检查 PyEIT 的源码/API，重点看：

- `protocol.create(...)`
- 协议中的 `ex_mat` / `meas_mat`
- JAC / BP / GREIT 等重建器对输入 `v0`, `v1` 的格式要求
- PyEIT 期望的是“测量电压差向量”还是“电极绝对电势”

请不要直接把 16×16 电势矩阵丢给 PyEIT。

你需要做的核心工作是：
1. 从 Excel 读取每一组激励下 16 个电极的电势向量 `p_i`
2. 根据 PyEIT 协议定义的测量对 `meas_mat`，把每一组电势向量转换成测量电压差
3. 拼接得到基线测量向量 `v0`
4. 拼接得到异物测量向量 `v1`
5. 用 `v1 - v0` 做差分重建

也就是说，如果某一组激励下某个测量对是 `(m, n)`，则该测量值应构造成：

`V_meas = p_i[m] - p_i[n]`

这里的 `m, n` 必须和 PyEIT 协议里的测量定义一致，而不是随便选。

如果 PyEIT 的标准协议不适合当前数据，请你修改协议或自己构造兼容的测量向量，但必须在代码注释里清楚解释原因。
